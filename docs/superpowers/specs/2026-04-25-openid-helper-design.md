# 获取当前 OpenID 临时入口设计

## 问题

当前项目已经引入 `WECHAT_ADMIN_OPENIDS` 管理员白名单，但用户还没有一个现成入口在真机调试时获取自己的 OpenID，因此无法完成第一位管理员的初始化配置。

## 目标

在不改变现有管理员绑定和通知逻辑的前提下，增加一个临时的“获取我的 OpenID”入口，方便在微信开发者工具真机调试时直接拿到当前微信对应的小程序 OpenID。

## 选定方案

采用“通知设置页增加按钮 + 后端增加临时接口”的方式：

- 在 `miniprogram/pages/notify/` 页面增加一个按钮；
- 点击按钮后前端调用 `wx.login()` 获取 `code`；
- 后端新增一条临时接口，用现有 `jscode2session` 流程换取当前用户 `openid`；
- 前端弹窗展示完整 OpenID，并在成功后写入剪贴板，便于用户直接粘贴到环境变量配置；
- 不修改现有管理员白名单判断与绑定通知接收人逻辑。

## 方案对比

### 方案 1：通知设置页按钮 + 临时接口（推荐）

- 优点：真机操作最直观；不需要看日志；复用现有登录换取 OpenID 的能力
- 缺点：会短暂增加一个仅用于初始化的接口，后续需要删掉

### 方案 2：仅打印后端日志

- 优点：改动少
- 缺点：需要再去云托管日志里找 OpenID，操作路径长，不适合当前需求

### 方案 3：直接放开管理员接口

- 优点：不需要新入口
- 缺点：会削弱当前权限控制，不符合现有安全方向

## 设计

### 前端

- 文件：`miniprogram/pages/notify/index.js`
- 文件：`miniprogram/pages/notify/index.wxml`

新增一个“获取我的 OpenID”按钮：

- 点击后调用现有 `wx.login()` 封装；
- 请求后端临时接口；
- 成功后调用剪贴板接口复制完整 OpenID，并弹窗展示完整 OpenID；
- 失败时复用现有错误提示逻辑。

按钮先常驻在通知设置页，等管理员初始化完成后再删除。本次范围内不增加额外配置开关。

### 后端

- 文件：`backend/server.js`

新增一条临时接口，例如：

- `POST /wechat/debug-openid`

行为：

- 接收 `{ code }`
- 校验 `code` 必填
- 复用现有 `exchangeCodeForOpenId(code)` 获取 OpenID
- 返回 `{ openid }`

这条接口只用于初始化查看当前 OpenID，不写入绑定数据，也不改变白名单。
这条接口在当前环境中先默认可用，不额外区分开发/生产；它只会返回“当前请求者通过本小程序登录换取到的 openid”，无全局读写副作用。拿到 OpenID 并完成管理员初始化后，由后续一次显式代码变更删除。

### 页面行为

当前 `GET /wechat/status` 在管理员白名单未配置或当前用户无权限时会返回错误。这个场景下：

- 通知设置页仍然显示“获取我的 OpenID”辅助按钮；
- 前端以 `GET /wechat/status` 返回 **HTTP 403** 作为稳定的“管理员尚未初始化/当前用户无权限”判断信号，不依赖错误文案；
- 为了支持这个分支判断，前端请求层需要把失败响应的 `statusCode` 暴露给页面逻辑，供通知页精确识别 403；
- 状态卡片不展示成功状态；
- “绑定当前微信为通知接收人”和“授权下一次新订单提醒”按钮在该状态下隐藏；
- 页面展示简短初始化引导文案，提示用户先获取 OpenID 并配置 `WECHAT_ADMIN_OPENIDS`；
- 页面允许用户先完成 OpenID 获取，再去配置 `WECHAT_ADMIN_OPENIDS`。

如果 `GET /wechat/status` 失败原因不是 403（例如网络失败、微信凭据缺失、服务异常）：

- 页面继续展示现有错误提示；
- 仍显示“获取我的 OpenID”按钮；
- 不把这类错误误判成管理员初始化态。

## 数据流

1. 真机进入“通知设置”页
2. 点击“获取我的 OpenID”
3. 前端调用 `wx.login()` 获取 `code`
4. 前端请求 `POST /wechat/debug-openid`
5. 后端调用微信 `jscode2session`
6. 后端返回当前用户 `openid`
7. 前端弹窗展示完整 OpenID，用户复制后填入 `WECHAT_ADMIN_OPENIDS`

## 错误处理

- 若未获取到 `code`，前端提示“未获取到登录 code”
- 若后端缺少 `WECHAT_APP_ID` 或 `WECHAT_APP_SECRET`，返回明确错误
- 若微信接口返回异常，沿用现有错误提示链路展示
- 临时接口失败时，后端沿用现有接口风格，返回对应 HTTP 状态码和 `{ "error": "..." }` 结构
- 若剪贴板写入失败，前端仍展示完整 OpenID，并提示用户手动复制

## 验收

- 通知设置页出现“获取我的 OpenID”按钮
- 真机点击后能弹出完整 OpenID
- 真机点击后优先自动复制到剪贴板；若复制失败，仍能看到完整 OpenID
- 拿到 OpenID 后可配置进 `WECHAT_ADMIN_OPENIDS`
- 现有管理员绑定、订单查看、订阅授权流程不受影响
